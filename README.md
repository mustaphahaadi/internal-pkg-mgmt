## 🚀 Quick Start (Phase 1)

### Prerequisites
- Docker + Docker Compose installed

### Run it
```bash
docker-compose up --build
```

### Install a package on a client
```bash
# Exec into any client container
docker exec -it web-server bash

# Install a package from the local repo
dnf install --disablerepo="*" --enablerepo="yum_local" -y wget
```

### Add a new package to the repo
```bash
./scripts/add-package.sh /path/to/your.rpm
```

### Browse the repo
Open http://localhost:8080/repos/yum_local/ in your browser.

## 📚 What I Learned
### Phase 1
In Phase 1, I learned how YUM repositories work under the hood — a repo is simply a directory of RPMs plus metadata generated by createrepo that tells DNF what packages are available and how to install them. I built a multi-container environment using Docker Compose, learning how to write Dockerfiles, mount volumes, use environment variables, and connect containers over a shared bridge network where they resolve each other by hostname. I configured nginx not as a traditional web server but as a file server using autoindex on, serving the RPM directory over HTTP so client containers could use it as a baseurl in their .repo config files. I wrote bash scripts using heredocs and variables to automate repo configuration on each client, and learned how to target a specific repo during install using --enablerepo / --disablerepo flags. The biggest insight was seeing how all these pieces — Linux package management, container networking, HTTP file serving, and shell scripting — connect together to simulate a real air-gapped enterprise environment.
